Dokumentation

Mit dem DMNMGR können DMN Dateien bearbeitet und Git Repositories verwaltet werden. DMN dient der umfangreichen Konfiguration von Anwendungen und dieses Tool soll einen universal Konfiguratpr für die Applikationslandschaft darststellen.

Der Data-Modeler

Im ersten Tab findet sich der Data-Modeler. Mit diesem Tool kann das Datenmodell auf dem die Decision basiert entworfen werden.

Einfach ein neues Modell anlegen

Und Eigenschaften hinzufügen

Neben den Eigenschaften kann der Datentyp definiert werden. Mit dem Button rechts neben dem Datentyp können einzelne Eigenschaften wieder entfernt werden.

Datentypen:

String Eine einfache Zeichenkette
Integer Eine Ganzzahl (z.B. 1)
Number Eine Gleitkommazahl (z.B. 1,234)
Enumeration Ein Wertbereich. Identisch zu String jedoch können bestimmte Werte vorgegeben werden. Die vorgegebenen Werte werden auch in die DMN übernommen und stehen dort zur Auswahl wenn ein entsprechendes Attribut verwendet wird.
date Ein Datum in ISO Notation (z.B. 2019-04-01)
Datetime Ein Datum mit Zeitangabe in ISO Notation (z.B. 2019-04-01T12:12:12.000Z)
boolean True oder False
array Eine Liste beliebiger Elemente. Kann alle Datentypen erneut enthalten.
Object Eine Datenstruktur die die Verschachtelung ermöglicht. Enthält beliebige weitere Eigenschaften mit beliebigen Datentypen in einer neuen tieferen Ebene.

Mit dem Data-Modeler können beliebige Baumförmige Datenstrukturen definiert werden. Diese sollten immer dem Format entsprechen, in dem Daten für die Decision erwartet werden. Die Baumstruktur hilft Datenbeziehungen darzustellen und gewährleistet Übersicht.

Und wozu?

Das Eingabedatenmodell wird benötigt um eine strukturierte Oberfläche zur Simulation komplexer Eingabedaten zu erzeugen. Dieses Datenmodell wird zur Angabe von Simulations- oder Testdaten verwendet.

Der Editor

Der DMN Editor im Bereich Dein Modell ist der original Camunda DMN Editor. Der DMNMGR erweitert dessen Modellierfunktion nur um Komfortfunktionen um Entscheidungen zu testen und zu simulieren sowie Anpassungen an DMN Dateien direkt in Git Repositories durchzuführen.

GIT

GIT wird verwendet um Konfiguration zu einem Teil des Entwicklungsworkflows zu machen. DMNMGR hat eine GIT Integration die es ermöglicht DMN Konfigurationen und Regeldefinitionen direkt wieder in ein Code Repository zu pushen.

Einstellungen

Um GIT nutzen zu können, müssen ein paar notwendige Einstellungen vorgenommen werden. Diese Einstellungen finden sich in der GIT Sektion der Einstellungen (STRG + K).

Benutzername Der Klarname unter dem Commits vorgenommen werden.
Email Die Email des verwendeten GIT Accounts.
Private Key Ein SSH Schlüssel, der Authorisiert ist auf das entsprechende GIT Repository zuzugreifen.
Public Key Das Public Pendant zu oben genanntem privatem SSH Schlüssel.

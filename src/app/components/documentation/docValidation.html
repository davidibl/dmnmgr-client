DMN Dateien können mit Hilfe der Funktion DMN Validieren validiert werden. Die Funktion findet sich im Hauptmenü unter DMN -> DMN Validieren. Hierbei wird das DMN XML an einen Webservice gesendet der mit Hilfe einer Validation-Engine einige Checks durchführt.

Validierungen:

Shadowed Rules

Dieser Validator prüft ob Regeln durch Regeln verdeckt werden. Dieser Check zeigt beispielsweise in Tabellen mit "Hit Policy: First" ob eine Regel niemals getroffen werden kann weil eine vorhergehende Regel alle Kriterien bereits abdeckt.

Conflicting Rules

Dieser Validator findet in Tabellen mit "Hit Policy: First" jene Regeln die andere Regeln verdecken. Zudem werden Regeln entdeckt die in Tabellem mit "Hit Policy: Unique" stets beide getroffen werden können. So können Fehler bei eindeutiger Regeldefinition die nicht offensichtlich sind gefunden werden.

Connected Requirements

Dieser Validator gibt alle Elemente eines Requirements Graph an, die nicht verbunden sind. Ein Requirements Graph sollte immer aus einem Zusammenhängenden Baum bestehen.
Diese Regel erzeugt nur ein Warning, da sie unzusammenhängende Elemente in einem DRD nicht notwendigerweise Fehler darstellen.

Aggregated Outputs

Dieser Validator prüft ob die Aggregatsfunktionen, die in Output Expressions verwendet werden, mit den gegebenen Datentypen tatsächlich möglich sind. So können beispielsweise nur Zahlen addiert werden.

Duplicated Rule

Dieser Validator findet doppelte Regeln.

Input/Output Type Declaration

Dieser Validator prüft ob bei allen Input und Output Spalten ein Datentyp angegeben wurde. Fehlt ein Datentyp wird die Tabelle als fehlerhaft markiert.

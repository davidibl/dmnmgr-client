GIT

GIT wird verwendet um Konfiguration zu einem Teil des Entwicklungsworkflows zu machen. DMNMGR hat eine GIT Integration die es ermöglicht DMN Konfigurationen und Regeldefinitionen direkt wieder in ein Code Repository zu pushen.

Einstellungen

Um GIT nutzen zu können, müssen ein paar notwendige Einstellungen vorgenommen werden. Diese Einstellungen finden sich in der GIT Sektion der Einstellungen (STRG + K).

Benutzername Der Klarname unter dem Commits vorgenommen werden.
Email Die Email des verwendeten GIT Accounts.
Private Key Ein SSH Schlüssel, der Authorisiert ist auf das entsprechende GIT Repository zuzugreifen.
Public Key Das Public Pendant zu oben genanntem privatem SSH Schlüssel.

Die integrierten GIT Funktionen

Alle GIT Funktionen finden sich im Hauptmenü unter Repository. Im Folgenden werden die einzelnen Funktionen beschrieben

Repository klonen Dieser Befehl öffnet den Klonen Dialog. Um ein GIT Repository zu klonen (auf die eigene Festplatte kopieren um daran zu arbeiten) muss der ssh-Url des Repositorys und ein Zielordner eingegeben werden.
Der Dialog führt mit einem Wizard durch diesen Prozess. Sobald das Repository geklont wurde, kann an den enthaltenen DMN Dateien gearbeitet werden.
Status Der Menüeintrag Status zeigt stets den aktuellen Status des Repository in dem sich die aktuelle Datei befindet:
  • Nicht verbunden - Die Dateien sind in keinem Repository
  • Clean - Repository geöffnet und keine Änderungen vorhanden
  • Lokale Änderungen - Dateien im Repository wurden verändert und nicht committed
  • Detached - Ein zurückliegender Stand wurde ausgecheckt -> Keine Änderungen möglich
Branch Der aktuelle Branch. Standard ist Master. Wird ein neuer Branch angelegt wird dessen Name angezeigt.
Branch -> Neuer Branch Öffnet einen Dialog zum Anlegen eines neuen Arbeitszweigs (Branch). Innerhalb des Dialogs muss ein Branchname angegeben werden. Der neue Branch wird direkt ausgecheckt.
Änderungen committen Nur sichtbar wenn sich im aktuellen Repository Änderungen befinden. Öffnet einen Dialog um eine fachliche Beschreibung der Änderungen für die Änderungshistorie anzugeben. Ein Commit schreibt die Änderungen im Repository fest.
Änderungen rückgängig Diese Funktion macht alle nicht committeten (festgeschriebenen) Änderungen rückgängig.
Achtung! Keine Bestätigung wird hier verlangt!
Push zu Server Überträgt alle Commites in das Remote Repository auf dem zentralen GIT Server. So werden Arbeitsstände geteilt.
Pull vom Server Holt alle Änderungen die im Remote Repository vorliegen. Sollte nach jedem öffnen direkt zu Beginn gemacht werden.
